# 快速修复 HBuilderX 识别问题

## 立即尝试的步骤

### 步骤 1：重新识别项目

1. 在 HBuilderX 中，**右键点击项目根目录** `tool-ai-uniapp`
2. 选择 **`从项目管理器中移除`**（不要删除文件）
3. 然后：`文件` → `导入` → `从本地目录导入`
4. 重新选择 `tool-ai-uniapp` 目录
5. 等待 HBuilderX 自动识别

### 步骤 2：检查项目图标

识别成功后，项目图标应该是：
- ✅ **uni-app 图标**（不是普通文件夹图标）

如果还是普通文件夹图标，继续下一步。

### 步骤 3：手动设置项目类型

1. **右键点击项目根目录**
2. 选择 **`项目`** → **`项目设置`**（或 `属性`）
3. 查看项目类型，应该是 `uni-app`
4. 如果不是，尝试修改为 `uni-app`

### 步骤 4：检查关键文件

确保以下文件存在且正确：
- ✅ `manifest.json` - 应该有齿轮图标
- ✅ `pages.json` - 页面配置文件
- ✅ `App.vue` - 应用入口
- ✅ `main.js` - 入口文件

### 步骤 5：尝试运行

如果项目已正确识别：
1. **右键点击项目根目录**
2. 选择 **`运行`** → **`运行到小程序模拟器`** → **`微信开发者工具`**
3. 如果提示配置微信开发者工具路径，选择你的安装路径

## 如果还是不行

### 方法 A：创建新项目并复制文件

1. 在 HBuilderX 中：`文件` → `新建` → `项目`
2. 选择：`uni-app` → `默认模板` → `Vue 3`
3. 创建项目（例如命名为 `tool-ai-uniapp-new`）
4. 将原项目的所有文件复制到新项目（覆盖同名文件）
5. 运行新项目

### 方法 B：检查 HBuilderX 版本

1. 查看 HBuilderX 版本：`帮助` → `关于`
2. 如果不是最新版，更新到最新版本
3. 下载地址：https://www.dcloud.io/hbuilderx.html

### 方法 C：使用微信开发者工具直接打开

如果 HBuilderX 一直无法识别，可以：

1. 确保项目根目录有 `app.json` 文件（已创建）
2. 打开微信开发者工具
3. 点击 `导入项目`
4. 选择 `tool-ai-uniapp` 目录
5. 填写 AppID
6. 点击 `导入`

**注意**：这种方式功能可能不完整，建议还是使用 HBuilderX 编译。

## 验证清单

运行前检查：
- [ ] manifest.json 存在且有齿轮图标
- [ ] pages.json 存在
- [ ] App.vue 存在
- [ ] main.js 存在
- [ ] pages 目录存在
- [ ] 项目图标是 uni-app 图标（不是文件夹图标）
- [ ] 右键项目可以看到 `运行` 菜单

✅ 右键项目有 `运行` 菜单  
✅ 可以正常编译到微信小程序
