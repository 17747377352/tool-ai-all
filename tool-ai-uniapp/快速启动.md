# 快速启动指南

## 前置条件

1. ✅ 已安装 HBuilderX（推荐）或微信开发者工具
2. ✅ 后端服务已启动（默认端口 8080）
3. ✅ 数据库已初始化

## ⚠️ 重要提示

**uni-app 项目必须通过 HBuilderX 编译后才能运行！**

如果直接用微信开发者工具打开源码目录，会提示"未找到 app.json"错误。

**正确方式**：使用 HBuilderX 编译运行（见下方步骤）

---

## 3步快速启动

### 步骤1：配置 AppID

打开 `manifest.json`，修改第39行：

```json
"appid": "你的微信小程序AppID"
```

> 如果没有 AppID，可以使用测试号，或使用微信开发者工具的"测试号"

### 步骤2：配置后端地址

打开 `common/utils/request.js`，修改第4行：

```javascript
const baseUrl = 'http://localhost:8080'; // 确保与后端服务地址一致
```

> 如果后端运行在其他地址或端口，请相应修改

### 步骤3：运行项目

**方式A：使用 HBuilderX（推荐）**

1. 用 HBuilderX 打开 `tool-ai-uniapp` 目录
2. 点击顶部菜单：`运行` → `运行到小程序模拟器` → `微信开发者工具`
3. 首次运行会提示配置微信开发者工具路径，选择你的微信开发者工具安装路径
4. 等待编译完成，微信开发者工具会自动打开

**方式B：使用微信开发者工具（需要先编译）**

⚠️ **注意**：不能直接用微信开发者工具打开源码目录！

1. 先用 HBuilderX 编译项目（见方式A）
2. 编译后会在 `unpackage/dist/dev/mp-weixin` 目录生成微信小程序代码
3. 在微信开发者工具中导入编译后的目录
4. 或使用 HBuilderX 的"运行到微信开发者工具"功能（推荐）

## 调试设置

在微信开发者工具中：

1. 点击 `设置` → `项目设置`
2. 勾选：
   - ✅ **不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**
   - ✅ **启用调试**

## 常见问题

### Q: 编译失败？
A: 检查文件路径和 `pages.json` 配置

### Q: 网络请求失败？
A: 
- 检查后端服务是否启动
- 检查 `request.js` 中的 `baseUrl` 是否正确
- 在微信开发者工具中勾选"不校验合法域名"

### Q: 登录失败？
A: 
- 检查 `manifest.json` 中的 AppID
- 检查后端微信配置（appid、secret）

### Q: 页面空白？
A: 打开控制台查看错误信息
